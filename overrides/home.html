AI tools ship code fast. Architecture erodes faster.

Every Copilot session, every Cursor tab-complete, every Claude fix: code gets added. But nobody checks whether the codebase still makes structural sense across files. Patterns fragment. Layers leak. Helpers multiply. Linters check style. Type checkers check types.

Drift fills that gap — deterministically, on every commit.

Problems every vibe-coder recognizes
MDS detects

You tab-complete get_current_user() — and it already exists in two other files

Three copies, three names, three slightly different JWT decodings. One returns None on failure, one raises, one returns the raw dict. Drift finds the clones across your codebase.

PFS detects

Four endpoints, four chat sessions, four incompatible error contracts

One raises HTTP 409. One returns 200 with {"error": ...}. One silently passes. One has no handling at all. Drift flags the inconsistency across the module.

AVS blocks

You prompt "just make it work" — the AI imports the DB model directly in the API route

Tests pass, the feature ships. But after 30 AI-prompted features, the layer boundary is gone. drift diff --staged-only blocks it before the commit lands.

BAT tracks

Mypy complains, you prompt "fix this" — the AI adds # type: ignore

Fine once. After 50 times, the type checker protects nothing. Drift tracks bypass density so you can see it growing before it's too late.

If you recognized even one of these — drift will find more in your codebase. Every scenario is a real pattern drift detects deterministically, with file locations, severity scores, and a concrete fix path. See the full code examples →

Why not just a linter

Cross-file, not single-file

Linters check files in isolation. Drift detects patterns that only emerge across module boundaries.

Deterministic, not probabilistic

No LLM, no cloud, no randomness. Same repo, same commit — same results. Every time.

Structural, not syntactic

AST-level analysis finds semantic near-duplicates that text-based clone detectors miss.

Scored, not binary

Every finding has a severity score and trend. Track architecture health across commits, not just pass/fail.

Agent-compatible output

JSON, SARIF, and MCP — structured output that coding agents can consume and act on directly.
